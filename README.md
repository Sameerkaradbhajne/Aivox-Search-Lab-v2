## 🚀 Core Research Modules

### 🖥️ Registry Explorer (Master-Detail)
A professional-tier data explorer that mimics the Windows Registry architecture. 
- **Hierarchical Tree**: Grouped by "Registry Hives" (Alphabetical Initials).
- **Data Grid**: A high-density table for deep inspection of `Reg_SZ` (Names), `Reg_DWORD` (Phone), and `Reg_BINARY` (Email).

### 🔍 Search Lab Engine (Binary v2.0)
The visualizer has been upgraded to a full laboratory dashboard:
- **Laboratory Metrics**: Real-time monitoring of **Latency** (ms), **Throughput** (R/s), and **Memory Load** (%).
- **Logic Trace**: Semantic, human-readable output of the algorithm's decision path.
- **Mission Control Feedback**: Integrated toast notification system for all I/O operations.

### 👤 Intelligence Suite (Developer Profiles)
Features a dedicated sidebar module for **Developer Intelligence**, displaying system authority and specialization (e.g., *Binary Logic Ops*).

---

## 🏗️ Technical Architecture

### 1. Dual-View State Engine
The app utilizes a centralized `viewMode` state to morph the interface between the **Directory Hub** (Card-based) and **Registry Explorer** (Tree-based) without losing search context or algorithm progress.

### 2. High-Performance Sorting
Data integrity is maintained via `useMemo` hooks, ensuring the contact repository is always binary-search-compliant ($O(\log N)$) through precise locale-aware sorting.
